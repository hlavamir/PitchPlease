# Version 1 — Arduino Mono-Color

> First hardware version: Arduino driving 2× mono-color LED strips via PWM, controlled by the vvvv patch over serial.

## Overview

V1 was the original PitchPlease hardware. Each of the two LED strips is controlled as a single RGB unit — the entire strip shows one color at a time. Color is set by the vvvv patch sending serial messages to the Arduino, which maps them to six PWM output pins (R/G/B × 2 strips).

## Hardware

- **Arduino Uno R3**
- 2× basic RGB LED strips (non-addressable, 4-pin: R, G, B, +12V) — each strip shows one color at a time
- Custom PCB (see below)

**Pin mapping:**

| Pin | Function |
|---|---|
| 3, 5, 6 | Strip 0 — R, G, B |
| 9, 10, 11 | Strip 1 — R, G, B |
| 8 | Status LED |

## Custom PCB

Designed in Fritzing, dated December 2019. Files in `v1/hardware/pcb/`. Includes: Fritzing project (`pitch_please_191222.fzz`), DWG, and exported etching layers (copper, mask, silk — top and bottom). A fabrication order subfolder (`order_191223/`) contains the Gerber files sent for manufacturing.

## Firmware

File: `v1/firmware/pitch_please_mono/pitch_please_mono.ino`

Serial baud rate: **57600**

Protocol: the vvvv patch sends 7-byte packets. Byte 0 is a command byte (0 = set LED absolute). Bytes 1–6 are the six channel values (R/G/B for each strip). Byte 255 is a confirmation/end-of-packet marker.

On startup, the Arduino runs a RGB test sequence across both strips.

## Communication with vvvv

vvvv sends color data over serial at 57600 baud. This is the same baud rate used in v2.0, so the vvvv patch side needed to be updated when switching to v2.2 (921600 baud).

## Status

Superseded by [[v2]] for vvvv-connected use, but one physical device still exists and is occasionally used standalone. On power-up the firmware runs an idle loop (the RGB test sequence on startup serves as a built-in animation), so the device can operate without a computer connected.
