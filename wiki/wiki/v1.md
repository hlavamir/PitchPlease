# Version 1 — Arduino Mono-Color

> First hardware version: Arduino driving 2× mono-color LED strips via PWM, controlled by the vvvv patch over serial.

## Overview

V1 was the original PitchPlease hardware. Each of the two LED strips is controlled as a single RGB unit — the entire strip shows one color at a time. Color is set by the vvvv patch sending serial messages to the Arduino, which maps them to six PWM output pins (R/G/B × 2 strips).

## Hardware

- Arduino (classic, with PWM pins 3/5/6/9/10/11)
- 2× 1m LED strips (mono-color, RGB via PWM — not addressable)
- Custom PCB (see below)

**Pin mapping:**

| Pin | Function |
|---|---|
| 3, 5, 6 | Strip 0 — R, G, B |
| 9, 10, 11 | Strip 1 — R, G, B |
| 8 | Status LED |

## Custom PCB

Designed in Fritzing, dated December 2019. Files in `pcbs/`. Includes: Fritzing project (`pitch_please_191222.fzz`), DWG, and exported etching layers (copper, mask, silk — top and bottom). A fabrication order subfolder (`order_191223`) is also present.

## Firmware

File: `arduino/pitch_please_mono/pitch_please_mono.ino`

Serial baud rate: **57600**

Protocol: the vvvv patch sends 7-byte packets. Byte 0 is a command byte (0 = set LED absolute). Bytes 1–6 are the six channel values (R/G/B for each strip). Byte 255 is a confirmation/end-of-packet marker.

On startup, the Arduino runs a RGB test sequence across both strips.

## Communication with vvvv

vvvv sends color data over serial at 57600 baud. This is the same baud rate used in v2a, so the vvvv patch side needed to be updated when switching to v2b (921600 baud).
