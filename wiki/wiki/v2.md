# Version 2 — WS2811 Addressable Strips

> Switch to addressable WS2811 LED strips, allowing per-segment color control. Three sub-versions: 2a (2 strips), 2b-intermediate (4 strips, Arduino R3, limited bandwidth), 2b (4 strips, Arduino R4, full independent control).

## Overview

V2 introduced WS2811 12V addressable LED strips, where every ~5 cm segment can be set to a different color independently. This unlocked proper audio-reactive visuals driven by the vvvv patch. Both sub-versions communicate via serial and use FastLED. Patterns are always symmetric — only half the LED data is transmitted per strip, and the firmware mirrors it.

---

## V2a — 2 Strips

**Firmware:** `arduino/pitch_please_w2811/pitch_please_w2811.ino`

**Serial baud rate:** 57600

| Parameter | Value |
|---|---|
| Strips | 2 |
| LEDs per strip | 38 (physical) |
| Unique positions received | 10 (mirrored by firmware) |
| Data pins | D7 (strip 0), D8 (strip 1) |
| LED type | WS2811, FastLED |

**Protocol:** vvvv sends `(2 strips × 10 positions × 3 bytes RGB) + 2` bytes per frame. The `+2` are a mode-switch byte and a confirmation byte (255). The firmware mirrors the 10 received positions to fill the full 38-LED strip symmetrically.

Has a built-in standalone idle mode (slow hue cycling) that activates when no serial data is received.

---

## V2b — 4 Strips, Arduino R4

**Firmware:** `arduino/pitch_please_w2811_4_channel_r4/pitch_please_w2811_4_channel_r4.ino`

**Serial baud rate:** 921600 (significantly higher than v2a — vvvv patch must use matching rate)

| Parameter | Value |
|---|---|
| Strips | 4 |
| LEDs per strip | 38 (physical) |
| Unique positions received | 19 (mirrored by firmware) |
| Data pins | D9, D10, D11, D12 |
| MCU | Arduino R4 |
| LED type | WS2811, FastLED |

**Protocol:** `(4 strips × 19 positions × 3 bytes RGB) + 2` bytes per frame.

The higher baud rate (921600 vs 57600) was needed to push data to 4 strips at adequate FPS. The vvvv patch was updated to match.

---

## V2b-intermediate — 4 Strips, Arduino R3

**Firmware:** `arduino/pitch_please_w2811_4-channel/pitch_please_w2811_4-channel.ino`

**Serial baud rate:** 57600

An intermediate step between v2a and v2b. Four WS2811 strips were connected, but the Arduino R3 couldn't receive enough data per frame at 57600 baud to drive all four strips independently. The workaround was to receive full color data for only 2 strips and duplicate those colors to the other 2. This was a bandwidth limitation of the hardware.

The problem was solved by switching to the Arduino R4, which supports 921600 baud — high enough to send independent color data for all four strips per frame.
