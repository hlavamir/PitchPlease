# Version 3 — ESP32 + DMX

> Current version: ESP32 receiving DMX512 input and driving 4× WS2811 LED strips. Two physical devices built, with hardcoded DMX start addresses 100 and 200.

## Overview

V3 is a major architecture shift. The microcontroller is now an ESP32 (AZ-Delivery USB-C variant) and the control protocol is DMX512 over RS-485, received via a MAX485 module on a custom PCB. The vvvv patch sends DMX from a computer via a USB-DMX interface (Enttec Pro or Enttec Open). Two identical devices have been built with different hardcoded DMX start addresses.

WiFi and Bluetooth are explicitly disabled in firmware to avoid interference with LED timing.

## Hardware

- **MCU:** ESP32 (AZ-Delivery, USB-C)
- **DMX interface:** MAX485 RS-485 transceiver module
- **LED strips:** 4× WS2811 12V, 140 cm each, 24 LEDs per strip
- **PSU:** 12V, 10A
- **PCB:** Custom design in Fritzing (`v3_esp32_dmx/hardware/250802_v3_esp32_dmx.fzz`), Gerber files exported August 2025 (`v3_esp32_dmx/hardware/250805 pcb/`)
- **Case:** 3D printed, designed in Rhino 3D. Two case iterations:
  - `v3`: parts dated Aug–Sep 2025 (`v3_esp32_dmx/hardware/3d print/`)
  - `v3-1`: revised parts dated Nov–Dec 2025 (`v3_esp32_dmx/hardware/3d print v3-1/`)

## DMX Channel Layout

Per device, starting at the device's DMX start address:

| Channels | Count | Purpose |
|---|---|---|
| +0 | 1 | Master dimmer |
| +1 | 1 | Mode |
| +2 … +5 | 4 | Per-strip dimmers (one per strip) |
| +6 … +77 | 72 | Group A: individual LED RGB (24 LEDs × 3 channels) |
| +78 … +80 | 3 | Group B |
| **Total** | **81** | channels per device |

## Connection to vvvv

The vvvv patch sends DMX to v3 devices via one of two paths:

- **Direct:** vvvv → Enttec Pro (USB-DMX) → XLR → device
- **Via ArtNet:** vvvv → local ArtNet → QLC+ → Enttec Open (USB-DMX) → XLR → device

Both devices can be addressed simultaneously since they have different start addresses (100 and 200).

## Deprecated Prototype

`standalone_esp32/` is an earlier prototype that explored a different architecture (audio analysis onboard the ESP32, web-based control UI over WiFi). It is deprecated and may be deleted. A future idea remains open: a WiFi bridge from a phone to vvvv (not directly to the ESP32) for remote light control — see Open Questions.
